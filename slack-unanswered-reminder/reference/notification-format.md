# 通知メッセージのフォーマット

SKILL.md のステップ7「通知送信」で本文を組み立てる際の**書式の正本**。通知を投稿する前に必ずこのファイルに従うこと。

**2段構成**で送る。**親メッセージ**（チャンネルに投稿し通知を発火）＋ その **スレッド返信**（詳細）。

## 親メッセージ（チャンネル投稿。通知を発火）

```markdown
<@{team_members[0]}> <@{team_members[1]}> ... <@{team_members[N-1]}>
未返信投稿リマインダー（YYYY-MM-DD）
{list_name} で N 件未返信
```

- 先頭は `team_members` 全員のメンション（config の順序のまま）
- 見出しは `list_name` と全項目の合計件数 N
- 詳細項目はここには載せない（スレッド返信に載せる）

## スレッド返信（親の thread_ts に投稿。担当者でグルーピング）

```markdown
<@{担当 User ID A}>
1. 【MM-DD HH:MM】<{メッセージリンク}|{タイトルを1行に圧縮}>
2. 【MM-DD HH:MM】<{メッセージリンク}|{タイトル}>

<@{担当 User ID B}>
1. 【MM-DD HH:MM】<{メッセージリンク}|{タイトル}>

未割当（担当者未設定・チーム外）
1. 【MM-DD HH:MM】<{メッセージリンク}|{タイトル}>
```

### 書式ルール

- **各項目は1行**。Slack のリンク記法 `<{メッセージリンク}|{タイトル}>` で**タイトルをクリック可能なリンク**にし、生 URL は本文に出さない。
- `【MM-DD HH:MM】` は**リンクの外**（`<...>` の前）に置き、テキストのまま残す。値は `メッセージリンク` の `ts` から復元した投稿時刻（JST）。
- タイトルは List 行の「タイトル」を1行に圧縮（改行は詰める）。タイトルに `|` が含まれても、`<URL|タイトル>` の**最初の `|`** が区切りになるだけで表示は崩れない。
- **タイトル内の `@xxx`（`<@U…>` / `<!subteam^…>` / `@表示名` などのメンション記法）はすべて削除する**。リンクテキストにメンションを残すと意図しない通知発火や表示崩れの原因になるため、タイトル圧縮時に取り除く（前後の余分な空白も詰める）。
- グループ見出し:
  - 本組メンバー → `<@担当 User ID>`（**本人のみメンション**）
  - 未割当（担当者空欄・チーム外） → `未割当（担当者未設定・チーム外）`（**メンションしない**）。常に最後、項目があるときのみ。
- グループの並びは `team_members` の config 順。**項目ゼロのグループは出さない**。
- 番号は**各グループ内で1から**振り直す（グループをまたいで連番にしない）。親メッセージの「N 件」は全項目の合計。
- **冒頭に全員メンションは入れない**（最初のグループ見出しから始める。担当者ごとに本人をメンションするため重複通知を避ける）。
- `reply_broadcast` は**使わない**。詳細が長く1通に収まらなければスレッド返信を複数に分割可（グループ境界で分けると読みやすい）。

## 表示イメージ（合成リンク形式）

```
未割当（担当者未設定・チーム外）
1. 【10-24 15:22】各疾患別の患者数を新規受診患者数で絞る事は可能でしょうか
```
（「各疾患別の…」部分が青いリンクになり、タップでスレッドに飛ぶ）
