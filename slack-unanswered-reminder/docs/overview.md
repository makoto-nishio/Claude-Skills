# Slack 未返信リマインダーのご紹介

**Slack の QA リストの「返信し忘れ」を、毎日自動でチェックしてお知らせするツールです。**

---

## 1. これは何？

QA リストには日々問い合わせが追加されますが、件数が増えると「まだ誰も返信していない項目」の見落としが起こりがちです。

このツールは、**人の代わりに毎日リストを巡回**して、

> 「完了していない ＋ チームの誰もまだ返信していない」

項目だけを拾い出し、チームのチャンネルにまとめて通知します。

### 導入前と導入後

| | 導入前 | 導入後 |
|---|---|---|
| 未返信の確認 | 各自がリストを目視でチェック | 毎日決まった時刻に自動チェック |
| 見落とし | 件数が多いと発生しがち | 未返信があれば必ず通知が届く |
| 確認の手間 | リスト全件に目を通す | 通知に出た項目だけ見ればよい |

---

## 2. 通知はこう届きます

未返信があった日だけ、チャンネルにこのような通知が届きます（**未返信ゼロの日は何も届きません**）。

```
未返信投稿リマインダー（2026-06-12）
コア機能チームQAリスト で 7 件未返信
```

くわしい内訳はこのメッセージのスレッドに、**担当者ごとにまとめて**表示されます。

```
@田中さん
1. 【06-08 10:21】○○病院様からのお問い合わせ
   （元のメッセージへのリンク）

@佐藤さん
1. 【05-13 10:04】△△クリニック様のご要望
   （元のメッセージへのリンク）

未割当（担当者未設定）
1. 【06-11 11:21】□□病院様の確認依頼
   （元のメッセージへのリンク）
```

- 自分の名前のところだけ見れば、**自分が対応すべき項目がすぐ分かります**
- リンクをクリックすると元のスレッドに飛べるので、そのまま返信できます

### 賢いところ

- チームの誰かが既にスレッドで返信していれば、通知されません
- チーム外の人でも、**スレッドで名指し（メンション）された人が返信していれば対応済み**とみなします
- つまり「本当に誰も手を付けていないもの」だけが通知されます

## 3. 導入に必要なもの

| 必要なもの | 補足 |
|---|---|
| Slack の QA リスト | 「完了」チェック欄と「メッセージリンク」列があるもの |
| 通知先チャンネル | チームの普段のチャンネルで OK |
| Windows PC ＋ Claude Code | チェック処理を実行する PC（チームで 1 台） |

---

## 4. 導入の流れ（3 ステップ）

### ステップ 1: 設定ファイルに 4 つの項目を書く

`config.yaml` というファイルに、次の 4 つを記入します。

```yaml
list_name: "コア機能チームQAリスト"   # ① 監視する QA リストの名前
notification_channel: C015EL8TN7R    # ② 通知先チャンネル
team_members:                        # ③ チームメンバー（メール or Slack メンバー ID）
  - yamada@example.com               #   会社のメールアドレスでそのまま書けます
  - sato@example.com
schedule:                            # ④ 実行スケジュール
  daily:
    time: "13:00"          # 毎日 13:00 に実行
    weekdays_only: true    # 平日のみ
```

> メンバーは**会社のメールアドレス**で書くのが一番かんたんです（自動で Slack ユーザーに紐づけます）。Slack のメンバー ID でも書けます。チャンネルや ID の調べ方は [README](../README.md) に手順があります。

### ステップ 2: 登録コマンドを 1 回実行する

PowerShell で次の 2 行を実行するだけです。

```powershell
cd C:\Users\<ユーザー名>\.claude\skills\slack-unanswered-reminder\scripts
.\deploy-task.ps1
```

### ステップ 3: 完了

以降は**毎日自動で動きます**。日々の操作は一切不要です。

---

## 5. 日々の使い方

**基本的に何もしなくて OK です。** 通知が届いたときだけ、次のように対応してください。

1. 通知のスレッドを開き、**自分の名前のグループ**を確認する
2. 各項目のリンクから元のスレッドに飛び、返信する
3. 対応が終わったら、QA リストの「完了」にチェックを入れる

返信するか「完了」にチェックを入れれば、翌日からその項目は通知されなくなります。

---

## 6. よくある質問

**Q. 通知が来ない日があるのはなぜ？**
未返信がゼロだった日は、何も送らない仕様です。通知がない＝すべて対応済み、ということなので安心してください。

**Q. 実行時刻や頻度は変えられる？**
変えられます。毎日決まった時刻のほか、「平日 9〜18 時の間、1 時間ごと」のような設定も可能です。設定ファイルを書き換えて登録コマンドをもう一度実行するだけです。

**Q. 自分のチームでも使いたい**
QA リストと通知先チャンネルがあれば導入できます。上の「導入の流れ」の 4 項目を自チーム用に書き換えるだけです。お気軽にご相談ください。

**Q. PC の電源が入っていないとどうなる？**
実行用 PC がログオフ中はチェックが走りません（次回ログオン後のタイミングで補完実行されます）。

**Q. 通知は誰の名前で届くの？**
このツールは専用の Bot やアプリを使いません。通知は、**ツールを設定・運用している担当者本人の Slack アカウント名義**で届きます（その人の Slack 連携を通じて送信するため）。発信元を変えたい場合は、別のアカウントで連携し直す必要があります。

**Q. 急に通知が来なくなった**
Slack との接続が切れている可能性があります。管理担当者にご連絡ください（再接続は数分で完了します）。
