# Slack 停滞スレッドリマインドのご紹介

**Slack の QA リストの「停滞案件（一定営業時間が過ぎても未完了のもの）」を、毎日自動でチェックしてお知らせするツールです。**

> ℹ️ よく似た名前の別ツール **「未返信投稿リマインダー」（`slack-unanswered-reminder`）** とは別物です。あちらは「チームの誰も返信していない案件」を拾うもので、こちら（**停滞スレッドリマインド** `slack-unfinished-reminder`）は「投稿から一定の営業時間が過ぎても未完了の案件」を拾います。両方を同時に使うこともできます。

---

## 1. これは何？

QA リストには日々問い合わせが追加されますが、件数が増えると「対応が止まったまま放置されている項目」の見落としが起こりがちです。

このツールは、**人の代わりに毎日リストを巡回**して、

> 「完了していない ＋ 投稿（スレッド作成）から一定の "営業時間"（初期設定は 48 営業時間＝2 営業日）以上たっている」

項目だけを拾い出し、チームのチャンネルにまとめて通知します。

> **「48 営業時間」って？** 問い合わせが投稿された直後はまだ対応する時間があるので通知しません。投稿から一定時間（初期設定は 48 営業時間＝2 営業日）たっても未完了のままの項目だけを「停滞」として拾います。**時間は "営業時間" で数えるので、土日はカウントされません**（祝日は対象外）。この時間は設定で変えられます（例: 24 営業時間＝1 営業日に短縮など）。

### 導入前と導入後

| | 導入前 | 導入後 |
|---|---|---|
| 停滞案件の確認 | 各自がリストを目視でチェック | 毎日決まった時刻に自動チェック |
| 見落とし | 件数が多いと発生しがち | 停滞案件があれば必ず通知が届く |
| 確認の手間 | リスト全件に目を通す | 通知に出た項目だけ見ればよい |

---

## 2. 通知はこう届きます

停滞案件があった日だけ、チャンネルにこのような通知が届きます（**ゼロの日は何も届きません**）。

```
⚠️ 停滞スレッドリマインド（2026-06-12）
@田中さん @佐藤さん @鈴木さん

48営業時間以上経過したスレッド: 7件

コアメンバーが担当しており、スレッド作成から48営業時間（2営業日）以上経過したスレッドです。

凡例:
🔴 20営業日以上経過
🟡 10営業日以上経過
⚪ 2営業日以上経過

担当者別の詳細はスレッドをご確認ください。
```

くわしい内訳はこのメッセージのスレッドに、**担当者ごとにまとめて**表示されます。見出しには件数、各項目には経過日数に応じた色（🔴🟡⚪）が付き、**タイトルがそのままリンク**になっています（1 項目 = 1 行）。

```
@田中さん（2件）
🔴 1. 【06-08 10:21】○○病院様からのお問い合わせ   ← タイトルがリンク
🟡 2. 【05-30 09:10】◇◇医院様の設定変更について

@佐藤さん（1件）
⚪ 1. 【05-13 10:04】△△クリニック様のご要望

未割当（担当者未設定）（1件）
🟡 1. 【06-11 11:21】□□病院様の確認依頼
```

- 自分の名前のところだけ見れば、**自分が対応すべき項目がすぐ分かります**
- 色を見れば、**どれだけ長く止まっているか**がひと目で分かります（🔴 ほど深刻）
- リンクをクリックすると元のスレッドに飛べるので、そのまま対応できます

### 賢いところ

- 投稿直後の項目は通知しません。**一定の営業時間（既定 2 営業日）が過ぎて初めて**「停滞」として拾います
- 経過時間は **営業時間ベース**なので、金曜の投稿が「土日をはさんだだけ」で急かされることはありません
- 経過日数に応じた色分けで、**長期間放置されている案件が埋もれません**

## 3. 導入に必要なもの

| 必要なもの | 補足 |
|---|---|
| Slack の QA リスト | 「完了」チェック欄と「メッセージリンク」列があるもの |
| 通知先チャンネル | チームの普段のチャンネルで OK |
| Windows PC ＋ Claude Code | チェック処理を実行する PC（チームで 1 台） |

---

## 4. 導入の流れ（3 ステップ）

### ステップ 1: 設定ファイルに 5 つの項目を書く

`config.yaml` というファイルに、次の 5 つを記入します。

```yaml
list_name: "コア機能チームQAリスト"   # ① 監視する QA リストの名前
notification_channel: C015EL8TN7R    # ② 通知先チャンネル
min_elapsed_business_hours: 48       # ③ 投稿後この営業時間（土日除外）以上たった項目だけ通知（既定 48＝2営業日）
team_members:                        # ④ チームメンバー（メール or Slack メンバー ID）
  - yamada@example.com               #   会社のメールアドレスでそのまま書けます
  - sato@example.com
schedule:                            # ⑤ 実行スケジュール
  daily:
    time: "13:00"          # 毎日 13:00 に実行
    weekdays_only: true    # 平日のみ
```

> メンバーは**会社のメールアドレス**で書くのが一番かんたんです（自動で Slack ユーザーに紐づけます）。Slack のメンバー ID でも書けます。チャンネルや ID の調べ方は [README](../README.md) に手順があります。

### ステップ 2: 登録コマンドを 1 回実行する

PowerShell で次の 2 行を実行するだけです。

```powershell
cd C:\Users\<ユーザー名>\.claude\skills\slack-unfinished-reminder\scripts
.\deploy-task.ps1
```

### ステップ 3: 完了

以降は**毎日自動で動きます**。日々の操作は一切不要です。

---

## 5. 日々の使い方

**基本的に何もしなくて OK です。** 通知が届いたときだけ、次のように対応してください。

1. 通知のスレッドを開き、**自分の名前のグループ**を確認する（🔴 が付いた項目から優先）
2. 各項目のリンクから元のスレッドに飛び、対応する
3. 対応が終わったら、QA リストの「完了」にチェックを入れる

「完了」にチェックを入れれば、その項目は通知されなくなります。

## 6. よくある質問

**Q. 通知が来ない日があるのはなぜ？**
停滞案件がゼロだった日は、何も送らない仕様です。通知がない＝停滞しているものはない、ということなので安心してください。

**Q. 実行時刻や頻度は変えられる？**
変えられます。毎日決まった時刻のほか、「平日 9〜18 時の間、1 時間ごと」のような設定も可能です。設定ファイルを書き換えて登録コマンドをもう一度実行するだけです。

**Q. 「何時間で通知するか」は変えられる？**
変えられます。設定ファイルの `min_elapsed_business_hours` の値（初期設定は 48 営業時間＝2 営業日）を書き換えるだけです。たとえば `24` にすれば 1 営業日で、`72` にすれば 3 営業日で通知対象になります。**数えるのは営業時間（土日を除外）**です。

**Q. 色（🔴🟡⚪）の意味は？**
投稿からの経過営業日の目安です。🔴＝20 営業日以上、🟡＝10 営業日以上、⚪＝2 営業日以上。長く止まっている案件ほど赤くなります。

**Q. 自分のチームでも使いたい**
QA リストと通知先チャンネルがあれば導入できます。上の「導入の流れ」の項目を自チーム用に書き換えるだけです。お気軽にご相談ください。

**Q. PC の電源が入っていないとどうなる？**
実行用 PC がログオフ中はチェックが走りません（次回ログオン後のタイミングで補完実行されます）。

**Q. 通知は誰の名前で届くの？**
このツールは専用の Bot やアプリを使いません。通知は、**ツールを設定・運用している担当者本人の Slack アカウント名義**で届きます（その人の Slack 連携を通じて送信するため）。発信元を変えたい場合は、別のアカウントで連携し直す必要があります。

**Q. 急に通知が来なくなった**
Slack との接続が切れている可能性があります。管理担当者にご連絡ください（再接続は数分で完了します）。
