# 通知メッセージのフォーマット（reference）

> SKILL.md の「実行ワークフロー」ステップ7（通知の送信）から参照される固定テンプレート。**送信前に必ず本ファイルの書式どおりに整形すること。**

## 全体構成

**2 段構成**で送る。**親メッセージ（見出し＋凡例）** をチャンネルに投稿し、その **スレッド返信** に担当者別の詳細を載せる。

## 親メッセージ（チャンネル投稿。通知を発火させる）

`{H}` は `min_elapsed_business_hours`（既定 48）、`{D}` は `{H}/24`（既定 2）。

```markdown
⚠️ 停滞スレッドリマインド（YYYY-MM-DD）
<@{team_members[0]}> <@{team_members[1]}> ... <@{team_members[N-1]}>

{H}営業時間以上経過したスレッド: N件

コアメンバーが担当しており、スレッド作成から{H}営業時間（{D}営業日）以上経過したスレッドです。

凡例:
🔴 20営業日以上経過
🟡 10営業日以上経過
⚪ {D}営業日以上経過

担当者別の詳細はスレッドをご確認ください。
```

## スレッド返信（親メッセージの thread_ts に対して投稿。担当者でグルーピング）

各グループ見出しに件数 `（N件）` を付ける。各項目は**1 行**にし、タイトルを Markdown リンク `[タイトル](メッセージリンク)` にする（先頭に色マーカー、日時、番号）。

```markdown
<@{担当 User ID A}>（{担当 A の件数}件）
🔴 1. 【MM-DD HH:MM】[{タイトル を 1 行に圧縮}](https://{workspace}.slack.com/archives/{channel_id}/p{ts_no_dot})
⚪ 2. 【MM-DD HH:MM】[{タイトル}]({メッセージリンク})

<@{担当 User ID B}>（{担当 B の件数}件）
🟡 1. 【MM-DD HH:MM】[{タイトル}]({メッセージリンク})

未割当（担当者未設定・チーム外）（{未割当の件数}件）
⚪ 1. 【MM-DD HH:MM】[{タイトル}]({メッセージリンク})
```

## 整形ルール

- **親メッセージ**は「⚠️ 停滞スレッドリマインド（日付）」の見出し → `team_members` 全員のメンション（config の順序のまま） → 件数行 → 説明文 → 凡例 → 締めの順
- 件数行の `{H}` と説明文・凡例の数値は `min_elapsed_business_hours` から動的に埋める（既定なら「48営業時間」「48営業時間（2営業日）」「⚪ 2営業日以上経過」）。凡例の 🔴20 / 🟡10 は固定
- **スレッド返信**は担当者ごとのグループ構成。**冒頭に全員メンションは入れない**（最初のグループ見出しから始める）。各グループ見出しは**末尾に件数 `（N件）`**:
  - 本組メンバー → `<@担当 User ID>（N件）`（**本人のみメンション**）
  - 未割当（担当者空欄・チーム外） → `未割当（担当者未設定・チーム外）（N件）`（**メンションしない**）。常に最後、項目がある場合のみ表示
- グループの並びは `team_members` の config 順。**項目ゼロのグループは出さない**
- **各項目は 1 行**。書式は `{色マーカー} {番号}. 【MM-DD HH:MM】[{タイトル}]({メッセージリンク})`:
  - **色マーカー**は行頭。経過営業日 `>=20`→🔴 / `>=10`→🟡 / `>=2`→⚪（既定しきい値なら全項目が ⚪ 以上）
  - **タイトルは Markdown リンク** `[タイトル](メッセージリンク)`（タイトルと URL を別行に分けない）
  - 番号は**各グループ内で 1 から**振り直す（グループをまたいで連番にしない）。親メッセージの「N 件」は全項目の合計
  - タイトルは List 行の「**タイトル**」を 1 行に圧縮する。整形ルール:
    - **メンションは削除する**: タイトル内の `<@U123456>` / `<@名前>` / `<!subteam^S…>`（`|名前` 付き含む） / `<!channel>` / `<!here>` および素の `@xxx` を**すべて取り除く**（誤通知防止＋可読性向上）。除去で生じる余分な空白・行頭の `cc` などの断片も整える
    - `[` `]` を含む場合は全角等に置換して、Markdown リンク記法（`[タイトル](URL)`）が壊れないようにする
  - `【MM-DD HH:MM】` は `メッセージリンク` の `ts` から復元した投稿時刻（JST）
  - リンクは List 行の `メッセージリンク` をそのまま使う（ts→link 変換は不要）
- スレッド返信は `reply_broadcast` を使わない。詳細が長く 1 通に収まらない場合は複数のスレッド返信に分割してよい（グループ境界で分けると読みやすい）
